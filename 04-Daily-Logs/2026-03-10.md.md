## 🔧 Back — Audit et bugs

### Fix architecture

- ❌ **Problème** : Pas de `main.go` dans le projet Chi → l'API ne pouvait pas démarrer
- ✅ **Fix** : Fichier `main.go` créé, API démarre correctement

### Bug `GET /api/me` → 401 Unauthorized

- **Cause** : Doublon de context key entre `middleware/authctx.go` et `reqctx/reqctx.go`
  - Le middleware écrit à la clé `1`
  - Le handler lit à la clé `0`
  - → Contexte jamais trouvé → 401
- **Statut** : Fix identifié, pas encore appliqué

```go
// Problème : deux clés différentes pour le même contexte
// authctx.go  → ctx.WithValue(r.Context(), contextKey(1), user)
// reqctx.go   → r.Context().Value(contextKey(0))

// Fix : aligner sur une seule clé partagée via reqctx
```

## 🧠 Apprentissages

### Technique

#### Régression Next.js 15.2.x — Pages Router
Next.js 15.2.x introduit une régression sur Pages Router. La version 15.1.7 est la dernière stable fiable pour notre configuration. À surveiller dans les changelogs officiels.

#### Context key collision en Go/Chi
Un pattern dangereux : définir des types `contextKey` identiques dans deux packages différents. Même si les types semblent similaires, Go les traite comme distincts → les valeurs ne se retrouvent jamais. Solution : centraliser la définition de la clé dans un seul package (`reqctx`) et l'importer partout.

#### Coexistence Tailwind v4 + Sass
Tailwind v4 peut coexister avec Sass sans conflit immédiat. Stratégie retenue : laisser les deux en place pendant la phase de migration de contenu, supprimer Sass lors d'une session dédiée UI pour éviter de bloquer le dev métier.
