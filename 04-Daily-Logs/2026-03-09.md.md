## 🧠 Apprentissages

### Technique

#### Go Micro découvert

https://github.com/micro/go-micro

**C'est quoi** : Framework Go pour construire systèmes distribués et microservices. Couche d'abstraction qui évite de réinventer la roue sur problèmes classiques architectures distribuées.

**Pertinence Midbec** : Si on part sur backend Go, Go Micro pourrait structurer l'API layer proprement. À évaluer vs Gin/Echo classique.

#### UnoPIM découvert

https://unopim.com/

**C'est quoi** : Product Information Management open-source basé sur Laravel. Couche d'abstraction au-dessus des données produits, avec API REST ou GraphQL intégrée.

**Pertinence Midbec** : Pourrait gérer les 500k+ pièces avec leurs références, compatibilités, images. Alternative à Ogasys ? Ou couche intermédiaire ?

**Question à creuser** : UnoPIM + Go API layer custom vs backend Go pur ?

#### Template RedParts — Architecture exemplaire

**Pattern injectable** :

```
api/
├── base/          Classes abstraites (contrats)
└── fake-api/      Implémentations fake
```

Pour passer à backend Go/Gin → créer nouvelle implémentation des classes abstraites, changer exports dans `api/index.ts`. **Zéro impact sur UI.**

**Redux bien structuré** : 9 slices (cart, wishlist, compare, shop, user, currency, quickview, mobileMenu, options, garage).

**Système SCSS custom** : 212 fichiers, 9 sections, thèmes multiples (`theme-red`, `theme-blue`, etc.). Changer couleur primaire = modifier `$theme-color: #e52727` dans `_variables.scss`.

### Process

**Méthodologie audit template** : Cursor AI + analyse manuelle = gain de temps énorme. Cartographie complète en quelques heures vs plusieurs jours manuellement.

### Architecture

**Décisions clés identifiées** :

|Question|Statut|
|---|---|
|Redux → Zustand ?|Oui, mais **phase dédiée** (pas simultanément avec migration contenu)|
|SCSS → Tailwind ?|**Migration progressive** possible — ne pas bloquer dev métier|
|Next.js 11 → upgrade ?|Planifier **après stabilisation contenu**|
|API fake → Go/Gin ?|Conserver fake pendant dev front, remplacer à phase intégration|
|`IVehicle` → `IAppliance` ?|**Oui** : `{ brand, type, model, serialNumber }`|
