## ✅ Réalisations — Matin

### 📰 Latest News — `BlockPosts.tsx`

**Audit**
- Composant `BlockPosts` déjà branché dans `page.tsx` avec `posts.slice(0, 3)`
- Fake data RedParts dans `blogPosts.ts` — conservée intentionnellement
- `BlogApi` abstract class déjà définie avec `getLatestPosts(limit)`
- Zéro hook TanStack Query pour le blog — branchement différé                  

**Fix largeur (pattern fullWidth)**
- Cause : `overflow: hidden` + marges négatives Slick se font rogner par le `containerClass`
- Solution : sortir `__carousel` hors du container, garder `SectionHeader` dedans
- Même pattern que `BlockProductsCarousel` avec `fullWidth`

**Fix alignement gauche**
- Cause : en sortant du container, le carousel perd les gutters horizontaux
- Solution : `px-3 sm:px-4 md:px-6 lg:px-8` sur le wrapper carousel

**🧠 Réflexe appris**
> Tout carousel Slick qui semble trop étroit →
> chercher `overflow: hidden` + marges négatives dans le SCSS
> avant de toucher quoi que ce soit d'autre.
> Le fix est architectural (sortir du container), pas CSS.

**Commit** : `fix(homepage): align BlockPosts carousel width with page layout`
