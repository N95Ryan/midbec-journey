## 💡 Découverte du jour

**L'API produits UnoPIM attend des filtres JSON en query string** — `categories IN [slug]` + `status = true`, avec pagination `page` / `limit`. Même pattern que la doc REST, pas de filtre simplifié côté URL.

**Les attributs produit sont nested par channel/locale** — le nom et l'image passent par `values.channel_locale_specific` et `values.common.image`, pas un champ plat. Fallback sur le SKU si le nom locale est absent.

**Réutiliser le pattern slice nil → `[]` côté Go** — comme pour l'arbre catégories jeudi, initialiser `out.Data = []Product{}` évite un `null` JSON qui casse le rendu React.

## 🧠 Ce que j'ai appris

**Principe** : Une page catégorie UnoPIM = trois appels parallèles (nœud, produits paginés, arbre pour breadcrumb/subnav). Server Component Next.js = endroit naturel pour ce `Promise.all`.

**Principe** : Pas de cache long sur les produits (contrairement aux catégories) — catalogue plus volatil, pagination directe vers UnoPIM acceptable pour l'instant.
