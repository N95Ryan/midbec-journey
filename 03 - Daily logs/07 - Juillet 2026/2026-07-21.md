### 💡 Découverte du jour

**Deux formats pour `invoiceDownloadURL` selon l'endpoint ERP.** Via `GET /orders/webuser`, l'ERP renvoie un **token GUID** (`[GUID masqué]`). Via `GET /orders/{orderId}`, la même facture expose une **URL complète** : `{ERP_BASE_URL}/attachments/files/{GUID}/download`. Le code ne gérait que le second cas.

**Les IDs ERP dépassent la précision JSON « safe ».** Des identifiants du type `[ID snowflake]` sont tronqués dès qu'ils passent par un `float64` (probe Go) ou un `number` JavaScript (front). Symptôme : **422** `one or more invoices are unavailable` avec un ID « proche » mais faux.

**Le login par compte prod fonctionne.** [Compte de test client masqué] — 6 factures listées, GUIDs présents, ZIP téléchargeable après correctifs.
